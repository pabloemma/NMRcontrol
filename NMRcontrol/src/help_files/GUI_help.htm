The NMR analyzer GUI

 

This is the help file for the NMR GUI

 

 

This is the current main interface to the NMR analyzer. In order to run the analyzer, a few things have to be  defined.

1.     The parameter file. This is where all the parameters are store. The current set is shown in the table and explained in the NMR analyzer help documentation. Right clicking on the filed will bring up a dialog to pick new file.

2.     Input files: Click on choose and it first lets you get to the right directory. Once you are there you can select as many files as you want. Those files will show up in “input files window”

3.     once you are done , hit run and it should start working. Currently the stop button does not do anything since I do not want to exit with a kill.



Converter: ReadNMR_short

In order to analyze the data created by the LANL NMR system, you first have to run the converter. You can do this by clicking on converter and select the directory where the CSV files are stored. The program, will check if there is already a ROOT directory

and if not will create a ROOT directory underneath. Then it will convert the csv file to root file and store the rootfile there. In order to run the converter you can also put on the command line the following statement:

ReadNMRshort -k /home/klein/macsmall_disk/NMRdataJan2018/15/ -i TEQ3598860010

where the -k specifies the input directory location and the -i the specific file. In this command TEQ3598860010.csv gets converted to TEQ3598860010.root.

QC_ana

This program takes as input a Qcurve file and produces a fit to the Qcurve. This program gets run by doing

Qcana Qcurve.root, where Qcurve.root is the full path of the Qcurve file. This should be usually in $NMR_ROOT/QC_files (see below). It will append to the file Qcurve.txt all the fit parameter needed for the Qcurve fit in NMRanan



NMRana

Now you are ready to analyze the root files with NMRana. Before you can continue you need to define NMR_ROOT. This is the location for the Qcurve, parameter and TE files. The program expects a directory structure

$NMR_ROOT/TE , $NMR_ROOT/TE_files, $NMR_ROOT/NMR_Par and $NMR_ROOT/QC_files.

  • TE_files: contains the TE files to analyzer

  • TE contains the Teglobal.csv. This is a comma separated outputfile which lets you look at some of the variables.

  • NMR_Par is the location of the parameter files., With those you control the analysis. See here

  • QC_files: This contains the analyzed Qcurve files plus the Qcurve.txt file. This contains the parameters determined form the fit in Qcana

Once you have set this up you are ready. First run an TE analysisi to get the calibration constant. From the commandline this would be

NMRana -d /home/klein/macsmall_disk/NMRdataJan2018/root/ TEQ3598815748.root -f /home/klein/macsmall_disk/nmrwork/NMR_Par/Jan15_coil2_pol.par, where the -d gives the data directory and the -f gives the parameter file to use.
